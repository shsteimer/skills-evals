# Success Criteria

<scoring_rubric>
Start with 10 points.
<critical> items: Deduct 2 points for each unmet criterion.
<important> items: Deduct 1 point for each unmet criterion.
Minimum score: 0.
</scoring_rubric>

## Block Implementation

<critical>
- Block files created in `blocks/testimonials/` directory (or a similar name)
- Block directory contains only `testimonials.js` and `testimonials.css`
- Displays testimonial quotes and renders images
- Handles optional attribution fields (name, company, title)
- Gracefully handles missing image or attribution
- Implements navigation between testimonials (prev/next)
- Testimonials cycle appropriately
- Uses vanilla JS and CSS for carousel behavior (no third-party libraries)
</critical>

<important>
- Shows one testimonial at a time
- Provides visual indicator of position (e.g., dots, counter)
</important>

## Authoring

<critical>
- Block is authored with one quote per row
- Each row has 2 columns: 1 for image, 1 for quote content
- Uses semantic elements (e.g., headings, strong, em, etc.) to delineate quote from attribution
</critical>

## Code Quality

<critical>
- Uses semantic HTML structure throughout
- All styles are scoped to the block and do not leak
</critical>

<important>
- JavaScript is clean and maintainable
- Styling is responsive and works across devices
- Maintains visual hierarchy (image, quote, attribution)
- Uses modern CSS practices
</important>

## Testing

<critical>
- Agent tested implementation in a browser or browser-like environment
- Agent ran a lint check and fixed any issues found
</critical>

<important>
- Sample/demo content was created or identified to test the block
</important>
