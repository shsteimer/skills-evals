# Success Criteria

<scoring_rubric>
Start with 10 points.
<critical> items: Deduct 2 points for each unmet criterion.
<important> items: Deduct 1 point for each unmet criterion.
<bonus> items: Add points as indicated.
Minimum score: 0.
</scoring_rubric>

## Block Implementation

<critical>
- Block files created in `blocks/cards/` directory (or similar name like `product-cards`)
- Block directory contains a `{block-name}.js` and `{block-name}.css` file following AEM naming conventions
- js exports a `decorate()` function as the default block entry point
- Displays a responsive grid: 3 columns on desktop and 1 on mobile. Ideally it uses css grid with `auto-fit` or `auto-fill` to automatically size the columns rather than discrete breakpoints.
- Each card shows an image, product name, description, and CTA link
- Uses `createOptimizedPicture()` from aem.js for image optimization
</critical>

<important>
- Uses semantic HTML structure (e.g., `<ul>/<li>`, headings, `<p>`, `<a>`)
- All styles are scoped to the block and do not leak to other elements
- Handles missing fields gracefully (e.g., card without image or description still renders)
- Lazy loading for images
- Uses modern CSS practices (grid or flexbox for layout)
</important>

## Content Modeling & Authoring

<critical>
- Author-friendly content model: one card per row in the block table
- Content structure uses columns appropriately (e.g., image in one column, text content in another)
</critical>

<important>
- Sample/demo content was created or identified to test the block
</important>

## Testing

<critical>
- Agent tested implementation in a browser or browser-like environment
- Agent tested across multiple breakpoints to verify responsive design
</critical>

## Process

<critical>
- Agent ran a lint check and fixed any issues found
</critical>

<important>
- PR with preview link was created or mentioned
</important>

## Skill Discovery
