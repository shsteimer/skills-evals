# Evaluation Improvements Plan

## Current state

### Eval skill (`.claude/skills/eval/SKILL.md`)
Pipeline: identify runs → reconstruct workspace → read check-results.json → resolve `[check: name]` criteria → launch subagent for judgment → assemble eval-result.json → clean up.

Subagent starts AEM dev server (`npx -y @adobe/aem-cli up --no-open --port {port} --html-folder drafts`), uses Playwright MCP for screenshots at desktop + mobile viewports. Only screenshots pages the agent created — doesn't fabricate test content. Parses agent conversation log to find pages the agent tested.

Key files:
- `.claude/skills/eval/SKILL.md` — skill definition
- `scripts/reconstruct-workspace.js` — workspace reconstruction (exported function + CLI)
- `scripts/utils/task-checks.js` — runs `tasks/{task-name}/checks.js` if it exists
- `scripts/utils/check-helpers.js` — shared async check utilities (file ops, lint, module introspection)
- `scripts/assemble-eval.js` — merges check + judgment results, computes scores, writes data files
- `scripts/parse-agent-log.js` — parses output.jsonl into structured summary

### Viewer tools (`tools/`)
Standalone HTML tools that load data via `?data=` URL param. Each tool lives in its own folder with `index.html`, `main.js` (ES module), and `styles.css`. Shared design tokens and utilities in `tools/shared.css` and `tools/shared.js`.

- `tools/eval-viewer/` — single-run eval results (score, insights, criteria, screenshots with lightbox). Shows agent, model, augmentation set name, iteration in meta line.
- `tools/conversation-viewer/` — renders parsed agent conversation from `output.jsonl`. Shows original prompt at top.
- `tools/diff-viewer/` — interactive diff view with file index, syntax-colored additions/deletions
- `tools/comparison-viewer/` — A/B comparison report (gate banner, aggregate metrics, per-task deltas)

Data files generated by `assemble-eval.js` into result folders:
- `eval-data.js` — eval result + run metrics (includes model, augmentationSetName, runSetId, iteration)
- `conversation-data.js` — parsed conversation events + metadata (includes prompt)
- `diff-data.js` — raw diff content + metadata

Usage: `tools/eval-viewer/index.html?data=../../results/<run-set>/<run>/eval-data.js`

Cross-links between viewers resolve sibling data files relative to the current `?data=` path.

### Run metadata
Augmentation files (`augmentations/*.json`) use named object format: `{name, augmentations: [...]}`. Legacy bare array format still supported.

`run-tasks.js` captures per-run metadata in each result's `task.json`:
- `augmentationSetName` — from the augmentation file's `name` field (null if no global augmentations or legacy format)
- `model` — from `${AGENT}_MODEL` env var (null if not set)
- `runSetId` — the timestamp shared by all runs in a set
- `iteration` — the iteration number within a run set
