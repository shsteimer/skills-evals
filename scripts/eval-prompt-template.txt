## Your Task

Assess the agent's overall performance and report on how well it completed the task.

Think carefully about both the overall agent output (code changes made, final response delivered to the user) and the steps the agent took to get there (tools used, decisions made, approach taken).

Consider all of this in context of the evaluation criteria specified above and the expected outcome.

## Guidelines

- Provide a holistic assessment, not just a checklist
- Be specific with examples from the artifacts
- Consider the criteria priorities but organize naturally
- Acknowledge when multiple approaches are valid
- Be constructive and fair in your judgment

## Output Format

Respond with a single JSON object. No markdown, no commentary, no code fences.

### JSON Schema

```json
{
  "score": <integer, earned points>,
  "maxScore": <integer, total possible points excluding bonus>,
  "overallSuccess": <boolean>,
  "summary": "<1-3 sentence justification of the score>",
  "strengths": ["<specific strength with artifact reference>"],
  "weaknesses": ["<specific weakness with artifact reference>"],
  "observations": ["<notable observations about approach or decisions>"],
  "criteriaChecks": [
    {
      "name": "<criterion name from the criteria>",
      "priority": "<critical|important|bonus>",
      "met": <boolean>,
      "points": <number, points earned for this criterion (0 if not met)>,
      "notes": "<specific evidence from artifacts>"
    }
  ]
}
```

### Scoring Rules

- Follow the scoring rubric defined in the criteria (start from 0, earn points for each met criterion).
- `score` is the sum of all `points` values from `criteriaChecks`.
- `maxScore` is the total possible points from critical and important items only (not bonus).
- Score and maxScore are integers (sum of point values from met criteria).
- `overallSuccess` is true when no critical criteria are unmet AND `score >= 0.8 * maxScore`.
- Every criterion from the evaluation criteria must appear in `criteriaChecks`.
- `points`: critical items earn 2 when met (0 when not), important items earn 1 when met (0 when not), bonus items earn their indicated value when met (0 when not).
